## 3. Single combined Docker image

### 3.1 Build strategy

Multi-stage Dockerfile at repo root produces **one** image that contains:
1. Python 3.10 + FastAPI + spaCy `en_core_web_lg` + Presidio.
2. Compiled static frontend assets in `/app/static/`.
3. Uvicorn serving both `/api/*` (and the original `/detect-pii-entities`, `/anonymize-*` paths kept for backward-compat) and the SPA at `/`.

```dockerfile
# Stage 1 — build frontend
FROM node:20-alpine AS web
WORKDIR /web
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ .
RUN npm run build              # outputs /web/dist

# Stage 2 — runtime
FROM python:3.10-slim
ENV PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends gcc curl \
 && rm -rf /var/lib/apt/lists/*
COPY backend/requirements.txt .
RUN pip install -r requirements.txt uvicorn \
 && python -m spacy download en_core_web_lg
COPY backend/ ./
COPY --from=web /web/dist ./static
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=5s --start-period=20s --retries=3 \
  CMD curl -fsS http://localhost:8000/health || exit 1
CMD ["uvicorn", "api:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 3.2 Backend changes (minimal)

- Add `/health` returning `{"status":"ok"}`.
- Mount static dir: `app.mount("/", StaticFiles(directory="static", html=True), name="spa")` — registered **last** so API routes win.
- Add `app.include_router(markitdown_router, prefix="/convert")` placeholder (returns 501 Not Implemented) — keeps the URL reserved.
- CORS already permissive; tighten to `localhost` defaults via env.

### 3.4 GitHub Actions — `.github/workflows/build-and-publish.yml`

- Triggers: `push` to `main`, `push` of tag `v*`, `workflow_dispatch`.
- Steps:
  1. `actions/checkout@v4`
  2. `docker/setup-qemu-action@v3` + `docker/setup-buildx-action@v3` (multi-arch).
  3. `docker/login-action@v3` → `ghcr.io` with `${{ secrets.GITHUB_TOKEN }}`.
  4. `docker/metadata-action@v5` → tags: `latest`, `<short-sha>`, `<git-tag>` (when applicable).
  5. `docker/build-push-action@v5` → platforms `linux/amd64,linux/arm64`, push `ghcr.io/senthilsweb/privacyshield`.
  6. `cache-from: type=gha, cache-to: type=gha,mode=max` to keep CI under ~6 minutes.

```yaml
permissions:
  contents: read
  packages: write
```

After first successful push, mark the GHCR package **public** in the GitHub UI (one-time).

## 5. Markitdown extension hook (future)

To make a future Markitdown integration a small additive change:

- **Backend**: empty router file `backend/markitdown_router.py` returning `501` for `POST /convert`. Wire it in `api.py` behind an env flag `ENABLE_CONVERT=false`.
- **Frontend**: `Convert` tab is implemented as a 4th `TabsTrigger` but rendered only when `import.meta.env.VITE_FEATURE_CONVERT === 'true'`. Default: hidden.
- **Dependency note**: `markitdown` is a Python package; document its addition in `docs/architecture.md` so the future PR knows which `requirements.txt` to update and which Dockerfile stage to extend.
- **No code paths** that would have to be refactored to enable it later — pure additive.

## 7. Risk & mitigation

| Risk | Mitigation |
|------|------------|
| shadcn/ui components depend on Next.js conventions (`"use client"`, `next/image`). | Components are largely framework-agnostic; remove `"use client"`, swap `next/image` for `<img>`. |
| Tailwind v4 vs v3 differences. | Stay on v3 for stability; revisit later. |
| GHCR package starts private. | One-time manual flip to public; document in README. |
| Multi-arch build time. | Use GHA cache (`cache-from/to: type=gha`). Expected ~5 min. |
| Removing blog content breaks external links. | Acceptable — never publicly linked. Note in CHANGELOG. |
| Single-image bloat (Python + spaCy model + node build artifacts). | Multi-stage build keeps node out of runtime; final image ~1.1 GB (spaCy `en_core_web_lg` is 600 MB). Acceptable for a self-hosted tool. |
